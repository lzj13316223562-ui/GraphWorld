## 1. 课题名称

暂定中文题目：

**面向 Agent 强化学习的自演化图世界模型研究**

- `GraphWorld-24/7: A Persistent Self-Evolving Graph World Model for Agent Reinforcement Learning`

## 2. 研究背景与意义

近年来，具身智能机器人、服务机器人和大模型驱动智能体在家庭、办公、商业与公共服务等场景中的应用不断扩展。机器人不再只需要完成单一导航任务或一次性抓取任务，而是需要长期处于开放环境中，持续面对任务到达、环境变化、角色切换和风险事件。传统以静态语义地图、单任务规划和有限 episode 为核心的问题设定，已经难以充分覆盖真实世界中“长期服务”“持续运行”“角色责任”“突发事件响应”等关键需求。

现有大量工作已经证明，图结构是表达空间拓扑、对象关系、功能关系和任务约束的有效形式。然而，当前主流 scene graph、semantic map 与 embodied planning 工作大多仍然建立在如下隐含前提之上：

- 世界在单个任务期间基本静止；
- 状态更新主要由机器人动作触发；
- 任务目标给定且在有限步内完成；
- 评价指标聚焦于单任务成功率或动作匹配。

与之相比，真实世界中的机器人工作方式更接近于一个持续运行的系统：

- 时间持续推进；
- 世界持续演化；
- 任务持续到达；
- 社会规则持续约束；
- 智能体必须不断判断下一步该做什么、不该做什么。

因此，本课题拟从“静态场景图”推进到“持续自演化图世界模型”，构建一种能够描述时间、外生事件、周期/随机/高优任务流、社会规则、长期奖励与持续 episode 的符号图世界，为长期 Agent 强化学习提供统一的环境建模与评测基础。

本课题的意义主要体现在三个方面。

第一，在理论上，课题试图把场景图从“环境表示”提升为“世界模型”，推动具身规划从静态任务走向长期持续决策。

第二，在方法上，课题拟提出一套推理-执行-反思的无限序列 Agent 规划框架，使智能体可以在长期运行中不断调整自身行为，而不是在完成一次任务后终止。

第三，在工程与应用上，课题拟构建一个可仿真、可训练、可评测、可扩展到真机的图世界环境，为家庭管家、酒店服务、商店巡检、园区后勤等长期服务机器人场景提供统一底座。

## 3. 国内外研究现状

### 3.1 语义地图与场景图构建

当前与图世界模型最接近的基础方向是语义地图与场景图构建。相关工作大致可分为四类。

第一类是在线增量语义建图。代表工作如 Hydra 和 Embodied Semantic Scene Graph Generation，强调机器人在探索过程中实时维护层次化场景图，并将空间、语义和拓扑信息组织在统一结构中。这类工作说明了图结构在机器人感知与地图构建中的有效性，但其重点仍在于“看到了什么”和“如何组织观测”，而非长期任务世界的持续演化。

第二类是开放词汇与功能型场景图。代表工作如 HOV-SG、Open-Vocabulary Functional 3D Scene Graphs，以及更近的 Social 3D Scene Graphs。此类研究已经从“物体在哪里”推进到“物体能做什么”“哪些部件可交互”“人与物体如何发生功能关系”。这些工作为后续图世界中的功能关系、控制关系、社会语义提供了重要基础，但整体上仍更多服务于 grounding、navigation 或当前任务相关推理。

第三类是面向规划的任务相关场景图。代表工作如 SayPlan、Taskography、Optimal Scene Graph Planning 和 MomaGraph。它们已经明确把图作为规划中间层，强调 task-relevant subgraph、graph-then-plan、state-aware 图更新等思想，说明图不仅可以表达场景，还可以直接服务规划决策。其中 MomaGraph 尤其强调 state-aware unified scene graph 与 task-oriented subgraph 的价值，是本课题的重要近邻工作。

第四类是更高层的世界模型化研究。典型代表包括 Generative Agents 和近年的 embodied world modeling 相关工作。这些研究已经开始讨论长期运行、多角色、持续记忆和世界持续变化的问题，说明“活世界”而非“静态任务舞台”具备明确研究价值。但多数工作采用自然语言记忆或弱结构化环境，尚未形成机器人可执行、可验证、可训练 RL 的显式图世界系统。

综上，现有地图与场景图研究已经证明：层次图、功能关系、状态更新和任务相关图都具有必要性；但仍缺乏把这些能力统一为“带时间演化与长期任务流的图世界模型”的工作。

### 3.2 房间拓扑与楼层布局生成

若将研究范围收缩到建筑内场景生成，特别是单层或多层楼宇中的房间布局，则相关工作更接近于房间拓扑生成、功能分区与空间规划问题。近年来，layout graph、bubble diagram、graph-conditioned floorplan generation 等方向逐渐发展成熟，已经能够从高层约束生成房间邻接关系、功能分区与连通结构。

这类工作表明：

- “先生成房间拓扑，再生成几何与细节”是一条合理且高效的路线；
- 房间邻接关系、public/private 分区、入口/走廊/垂直交通核等关键要素都可以通过图结构表达；
- 房间级拓扑约束能够作为后续对象生成、导航规划和任务模拟的稳定上层结构。

但这类研究的目标通常止步于“布局生成”本身。它们并不直接关心：

- 房间拓扑如何承载长期任务流；
- 房间与对象如何共同组成可执行世界状态；
- 世界在长期运行中如何随时间和外生事件演化。

因此，本课题将房间拓扑视为图世界模型的第一层，而不是最终目标。

### 3.3 任务规划、Agent 与长期运行

在任务规划方面，现有研究已经从传统 symbolic planning 逐步扩展到 LLM-based planning、vision-language-action models、long-horizon task decomposition 和 safety-aware planning。代表工作包括 SayCan、Inner Monologue、Code as Policies、SayPlan、LoHoVLA、AndroidWorld、SafePlan-Bench 和 VESTABENCH 等。

这些工作在长任务、多步推理和执行反馈方面均取得了重要进展，但仍然普遍具有如下特点：

- 任务目标预先给定；
- episode 长度有限；
- 任务完成后环境即停止或重置；
- 评测以单任务成功为核心。

从本质上看，这仍然是一种“执行-更新-执行”的有限回合范式。即使任务链更长，环境更复杂，研究对象也通常仍是“如何完成这一件任务”，而非“如何在一个持续运转的世界中长期担任角色”。

相比之下，本课题拟研究的问题更像是一个无限生存沙盒中的长期职责管理问题。智能体需要同时面对：

- 常规任务；
- 定期任务；
- 紧急任务；
- 随机事件触发任务；
- 资源、电量和上下文约束；
- 社会规范、权限、隐私和风险控制。

例如，当厨房油锅起火时，机器人应继续配送任务、立即前往灭火、寻求帮助，还是采取其他更保守的策略？这类问题无法通过单任务成功率充分刻画，必须在长期运行的图世界中从状态变化、任务耦合和后续效用中进行评价。

### 3.4 现有工作的不足与本课题切入点

综合以上分析，当前相关工作主要存在以下不足：

- 地图与场景图工作多数忽略时间和长期演化；
- 任务规划工作多数停留在有限 episode；
- 房间拓扑生成工作多数只关注布局本身；
- 长期 Agent 工作多数缺乏机器人可执行的显式图状态与规则系统；
- 评价体系多数仍聚焦于单任务完成，而非长期职责与服务效用。

本课题的切入点在于：提出一个**自演化图世界模型**，将房间拓扑、对象关系、时间推进、外生事件、任务流、社会规则、长期奖励与持续 episode 统一到一个可执行、可训练、可评测的符号图世界中。

## 4. 研究目标与拟解决的关键问题

### 4.1 总体目标

构建一套面向 Agent 强化学习的持续自演化图世界模型及其规划框架，使智能体能够在长期运行、持续变化的环境中，依据图状态进行推理、执行与反思，并逐步形成稳定的长期服务策略。

### 4.2 具体目标

1. 设计一种融合空间、时间、功能、规则与任务流的符号图世界表示。
2. 建立一套以房间拓扑为骨架、逐房间生成对象与关系的场景构建方法。
3. 设计包含时间、外生事件、周期/随机/高优任务流、社会规则、长期奖励和持续 episode 管理的运行引擎。
4. 提出推理-执行-反思的无限序列 Agent 规划框架。
5. 构建仿真 benchmark，并在仿真与真机上验证该图世界模型对长期决策学习的支撑作用。

### 4.3 拟解决的关键问题

1. 如何定义能够承载长期演化的图世界状态，而不是仅仅表示静态场景？
2. 如何将时间、事件和任务流以统一方式纳入图状态与状态转移之中？
3. 如何让房间拓扑、对象关系和角色职责共同构成长期任务环境？
4. 如何在无限或超长时域中设计合理的长期奖励，避免智能体投机取巧？
5. 如何评价一个长期运行机器人是否真正“担任好了角色”？

## 5. 研究内容

### 5.1 自演化图世界模型

本课题拟提出的图世界模型，不是简单的 scene graph，而是一个由多层节点、多类边和动态状态共同构成的持续演化系统。其图层次初步包括：

- 建筑层：building、floor、room、corridor、vertical core；
- 物体层：container、surface、tool、appliance、hazard source、resource；
- 角色层：robot、human、role、permission；
- 任务层：job、event、routine、emergency；
- 规则层：norm、priority、prohibition、temporal constraint。

图中的每个节点和关系都不仅具有静态属性，还应具备动态状态、时效性、生命周期与交互能力。世界状态既可以因智能体动作变化，也可以因时间推进和外生事件变化。

### 5.2 基于房间拓扑的场景生成

本课题拟采用“先拓扑、后填充”的场景生成主线。

第一步，先生成楼层房间拓扑。重点确定：

- 房间类型；
- 房间邻接关系；
- 入口与走廊组织；
- public/private 分区；
- 垂直交通核心。

第二步，基于房间功能逐房间填充物体、部件和局部关系，形成房间级功能子图。

第三步，在静态图上初始化时间、资源、角色职责与规则系统，构成可运行的世界初始状态。

### 5.3 持续运行世界引擎

图世界运行引擎将重点支持以下六个核心能力：

1. 时间；
2. 外生事件；
3. 周期/随机/高优任务流；
4. 社会规则；
5. 长期奖励；
6. 持续 episode 管理。

其中，时间推进负责驱动世界状态变化；外生事件负责模拟人类世界中的突发扰动；任务流负责让智能体长期处于职责管理状态；社会规则负责引入隐私、权限、礼仪、安全和角色责任等约束；长期奖励负责把单次行为影响映射到长期效用；持续 episode 管理则使环境不再在单任务完成后立即终止。

### 5.4 推理-执行-反思的 Agent 规划框架

本课题拟提出一套面向长期运行环境的 Agent 框架，主要包括三个过程：

- 推理：基于当前图世界状态、时间信息、任务流和角色职责判断优先级与风险；
- 执行：调用动作原语对图世界实施状态变换；
- 反思：根据执行结果、长期后果和历史上下文调整未来策略。

在记忆管理上，后续还将探索：

- 长期记忆：面向 RAG 检索与长期经验库；
- 中期记忆：面向上下文总结与任务簇压缩；
- 短期记忆：面向最近若干步执行、成功/失败反馈和临时计划维护。

### 5.5 训练目标与评价体系

本课题不再仅以“单任务是否成功”作为核心指标，而是希望建立长期职责导向的评价体系。初步考虑的评价维度包括：

- 常规任务完成情况；
- 紧急事件响应情况；
- 长期服务效用；
- 社会规则遵守情况；
- 世界健康度与环境风险控制；
- 资源利用率和稳定运行能力。

这一部分仍需进一步细化，是后续研究中的重点难点之一。

## 6. 技术路线

本课题拟采用如下技术路线。

第一，构建房间拓扑生成模块。先以单层房间拓扑为起点，生成房间级邻接图，再逐步扩展到多层建筑结构。

第二，构建逐房间对象生成模块。根据房间功能和规则模板，在房间内部生成对象、部件和局部交互关系。

第三，构建自演化图世界引擎。在现有静态图基础上增加时间推进、外生事件、任务流、社会规则、长期奖励和持续 episode 管理。

第四，构建 Agent 规划与训练接口。设计基于图状态的 observation、动作原语、状态转移和 reward 接口，使整个系统能够直接服务 RL 训练与长期决策研究。

第五，构建仿真与真机验证流程。先在符号仿真环境中验证图世界模型和长期策略学习效果，再逐步迁移到真实机器人平台。

整体流程可以概括为：

`房间拓扑生成 -> 逐房间对象填充 -> 图世界初始化 -> 时间/事件/任务流驱动运行 -> Agent 推理执行反思 -> 长期奖励与评价`

## 7. 拟创新点

### 7.1 提出融合时空观念的符号图世界模型

区别于现有主要关注静态环境与局部状态更新的 scene graph 研究，本课题拟把时间、生命周期和长期演化引入图表示之中，使图不再只是场景描述，而成为世界模型。

### 7.2 提出推理-执行-反思的无限序列任务规划架构

区别于传统 episode 式单任务规划，本课题拟面向无限或超长时域环境，研究智能体如何长期管理职责、任务、风险与资源。

### 7.3 提出面向长期 RL 的图世界 benchmark

区别于当前多以单任务成功率为核心的规划 benchmark，本课题拟构建一个能够支持长期奖励、持续 episode 和职责导向评价的图世界环境。

## 8. 实验方案

### 8.1 仿真实验

第一阶段重点开展符号仿真实验，不依赖 3D 感知和多模态输入，直接在图世界中训练和评估 Agent。初步考虑的典型场景包括：

- 住宅；
- 办公/酒店；
- 商店。

如条件允许，后续可扩展到医院和学校等更高规则密度环境。

实验重点包括：

- 图世界建模能力验证；
- 房间拓扑与对象生成合理性验证；
- 外生事件与任务流驱动下的长期决策验证；
- 长期 RL 策略学习效果评估；
- 不同 Agent 框架的对比实验。

### 8.2 真机实验

在后续阶段，可选取代表性长期服务任务进行真机验证，如：

- 定时配送；
- 巡检；
- 保洁与物资管理；
- 工具使用；
- 突发事件中断与恢复。

真机实验的重点不是复杂视觉建图，而是验证图世界模型和长期规划逻辑是否具备现实可迁移性。

## 9. 可行性分析

本课题具有较好的可行性基础。

第一，现有 `GraphWorld` 代码库已经具备场景图、房间生成、对象规则、符号执行和 benchmark 雏形，可为后续研究提供稳定底座。

第二，当前项目已经明确形成了“房间拓扑生成 + 房间内对象填充 + 状态更新”的静态流程，后续新增时间、事件和任务流具有较强工程连续性。

第三，本课题第一阶段以符号图世界和仿真验证为主，不依赖复杂的 3D 感知系统，有利于控制研究变量、快速开展实验并验证核心思想。

第四，当前相关文献已经充分证明 scene graph、task-relevant graph、state-aware graph、long-horizon agent 和 world modeling 等方向都具有研究价值，因此本课题的切入点具备较强的理论支撑。

## 10. 预期成果

本课题拟达到以下预期成果。

1. 形成一套自演化图世界模型的形式化定义与实现框架。
2. 形成一条以房间拓扑为骨架的图世界生成流程。
3. 构建一个支持长期 RL 的持续运行仿真环境与 benchmark。
4. 提出一套推理-执行-反思的长期 Agent 规划框架。
5. 完成仿真验证，并在条件允许时开展真机实验。
6. 产出论文、技术报告和可复用代码系统。

## 11. 研究计划

### 第一阶段：问题定义与图世界底座

- 明确图世界状态定义；
- 完成房间拓扑生成器与逐房间对象填充主线；
- 统一静态图表示与状态真值结构。

### 第二阶段：持续运行机制

- 增加时间、外生事件和任务流；
- 设计社会规则与长期奖励；
- 实现持续 episode 管理。

### 第三阶段：Agent 框架与仿真实验

- 实现推理-执行-反思框架；
- 构建 RL 环境接口；
- 开展长期仿真实验和基线比较。

### 第四阶段：系统完善与论文撰写

- 完善评价体系；
- 补充可解释案例分析；
- 整理论文与开题材料；
- 准备真机验证。

## 12. 当前仍待明确的问题

尽管主线已经较为清晰，但以下问题仍需继续深入：

- 世界状态的真值形式到底应采用纯图、事实集还是二者混合；
- 时间推进应采用离散 step、分钟级时钟还是事件驱动机制；
- 社会规则如何形式化进入图状态、动作约束和奖励函数；
- 长期奖励如何避免智能体只刷低风险任务；
- 如何定义角色职责与角色切换；
- 如何评价一个机器人是否真正“担任好了角色”；
- 长期、中期、短期记忆如何与图世界状态耦合。

## 14. 结语

本课题要解决的，不是“如何在一张静态场景图上完成一件任务”，而是“如何在一个持续自演化的图世界中，让智能体长期、稳定、合规地承担角色并做出决策”。如果该问题能够被较好地形式化并实现，它有望把具身智能研究从“单任务规划”推进到“长期世界建模与持续服务智能体”的新阶段。
