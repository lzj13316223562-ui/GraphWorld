\begin{abstract}
现有具身智能基准大多采用有限 episode 设定：系统给定一个初始状态、一个任务目标和一个终止条件，智能体的能力由是否在有限步内完成任务来衡量。然而真实服务机器人面对的不是一次性任务，而是持续被人类活动、环境时间和外生事件扰动的开放世界。本文提出 GraphWorld，一种面向长时程具身智能研究的持续自演化符号图世界模型。GraphWorld 将场景图从静态语义快照扩展为可运行的世界状态：房间、物体、设备、人和机器人被统一组织在层次化图中，节点状态与动态关系会随 NPC 日程、设备周期、自然过程和机器人动作持续更新。系统进一步提供局部感知、合法动作验证、事件日志、矩阵化评分和可复现实验流水线，使同一图表示能够支撑场景构建、状态追踪、任务生成、符号执行和长期评价。针对长程任务中语言模型容易陷入局部动作选择的问题，我们实现了技能条件化机器人 agent，将高层任务、可读技能阶段与低层合法动作候选相结合。家庭、医院和超市场景的初步实验表明，相比无机器人基线，引入机器人维护行为能够显著改善空间恢复、人类事件成功率和综合得分。GraphWorld 的核心目标不是定义一个固定任务集，而是提供一种可持续产生任务、持续积累状态、并可用于研究长期维护行为的图世界基础设施。
\end{abstract}

\section{引言}
当前 embodied AI 的一个基本假设常被默认接受：任务是有限的。导航、抓取、开关物体和语言条件任务通常被组织为一个 episode，环境给定起点和目标，智能体只需在终止前完成指定目标。这种设定便于评测，也推动了感知、导航和短程规划的发展。但真实家庭、医院、超市和办公室并不会在机器人完成一次任务后停止变化。衣服会再次变脏，垃圾会继续产生，杯子会被重新使用，药品和耗材会被消耗，顾客会移动商品，病人和护士会改变房间状态。机器人面对的不是 ``solving a task''，而是 ``living with an endless stream of tasks''。我们将这一设定称为无限任务问题。

无限任务问题改变了具身智能研究中的几个基本对象。首先，世界状态不能只被视为一张静态地图加少量临时变量，而必须是一个持续运行、持续被改写的状态过程。其次，任务不再只是外部标注的指令，而是由人类活动、物体状态、时间推进和机器人干预共同产生。再次，评价指标不能只关心一次任务是否成功，而要衡量机器人是否长期维持了世界的秩序、可用性和人类活动的顺畅程度。

场景图和语义地图为这一方向提供了重要基础。已有研究证明，结构化图表示可以支持语义定位、任务 grounding 和大规模规划~\cite{hydrarss2022,sayplan2023,taskography2022}。但多数方法仍把图作为当前时刻的环境描述，服务于有限任务中的感知或规划。另一方面，持续社会仿真和生成式角色系统强调世界与角色可以长期运行~\cite{generativeagents2023}，但它们往往依赖自然语言记忆或行为轨迹，而不是可供机器人动作验证和状态评分的显式符号图。

GraphWorld 试图连接这两个方向。我们的核心观点是：如果场景图足够明确地表达空间层次、物体可供性、控制关系、动态状态和时间上下文，它就不只是环境表示，而可以成为一个 persistent world substrate。GraphWorld 将图作为运行时世界模型，使同一套数据结构同时承担场景生成、状态演化、NPC 扰动、机器人动作执行和长期评价的职责。

本文的贡献包括三点。第一，我们提出一种持续自演化图世界模型，将楼层、房间、物体、设备、人类 NPC 和机器人统一纳入层次化符号图，并显式建模时间上下文和动态图状态。第二，我们实现了一个运行时引擎，将人类日程事件、设备周期、环境过程和机器人动作纳入同一状态转移流程，并通过矩阵化状态距离、空间关系距离和人类事件成功率评价长期表现。第三，我们构建了技能条件化机器人 agent，使语言模型负责高层语义选择，图引擎负责合法动作验证和技能阶段约束，从而更稳定地处理洗衣、倒垃圾、补给和清洁等长程维护任务。

\section{相关工作}
\subsection{场景图与语义世界表示}
场景图和语义地图已经成为具身智能中的重要中间表示。Hydra 等工作研究了实时 3D 场景图构建和优化~\cite{hydrarss2022}，SayPlan 将 3D 场景图用于大规模语言模型规划 grounding~\cite{sayplan2023}，Taskography 则从任务规划角度评价大规模 3D 场景图~\cite{taskography2022}。这些工作说明，图结构能够有效表达空间层次、对象关系和任务相关上下文。GraphWorld 继承这一结构化表示优势，但进一步强调图的运行时演化：节点状态和动态关系不是 episode 中的附属变量，而是世界持续变化的核心。

\subsection{长时程任务与语言模型规划}
SayCan、Inner Monologue 等方法将语言模型与机器人可供性、环境反馈或中间推理结合，用于长程任务分解和动作选择~\cite{saycan2022,innermonologue2022}。这类方法通常仍以给定任务为中心，即先有一条指令，再将其分解为可执行动作。GraphWorld 关注的是更靠前的一层：任务如何在持续世界中不断出现，以及机器人如何在没有固定终点的环境里持续维护状态。因此，我们不只需要规划器，还需要一个能产生扰动、暴露问题并记录恢复效果的世界底座。

\subsection{持续世界与社会仿真}
Generative Agents 等系统展示了长期运行的角色、日程和社会行为~\cite{generativeagents2023}。这类系统启发我们将 NPC 视为世界动力学的一部分，而非简单背景对象。但对机器人研究而言，仅有自然语言记忆并不足够：机器人需要知道门是否打开、杯子是否有水、衣服是否干燥、药品是否在正确位置，以及某个动作是否合法。因此，GraphWorld 采用显式图状态和规则化转移，把社会活动产生的扰动转化为可验证、可评分、可恢复的符号变化。

\section{GraphWorld 模型}
\subsection{持续图世界定义}
我们将 GraphWorld 定义为带时间推进机制的层次化符号图：
\[
\mathcal{G}_t=(\mathcal{V}, \mathcal{E}_s, \mathcal{E}_t, \mathcal{A}, \mathcal{S}_t, \mathcal{W}_t),
\]
其中 $\mathcal{V}$ 为节点集合，$\mathcal{E}_s$ 为相对稳定的结构边，$\mathcal{E}_t$ 为运行时动态边，$\mathcal{A}$ 为节点静态属性与可供性，$\mathcal{S}_t$ 为时刻 $t$ 下的节点状态，$\mathcal{W}_t$ 为全局世界上下文，包括 step、分钟级时间、日期、可见房间、事件日志和角色活动阶段。

节点类型包括 room、fixed object、movable object、control object、human 和 robot。边关系包括包含、邻接、支撑、控制和运行时位置关系。结构边描述房间连通、对象组成和设备控制等稳定信息；动态边由 parent、runtime relation 和运行时索引重建，用于表达 \texttt{in}、\texttt{on}、\texttt{at}、\texttt{near}、\texttt{held\_by} 和 \texttt{worn\_by} 等关系。这样的区分使房间拓扑和临时物体位置保持解耦，同时允许图在每一步被重新物化为完整 scene graph。

动态状态是图世界可演化性的核心。当前实现中的状态包括清洁、开闭、开关、湿度、折叠、腐烂、烧焦、填充度、设备剩余周期、植物生命值、人类心情和当前活动等。状态既可由机器人动作改变，也可由 NPC 事件、设备周期和环境时间改变。例如洗衣机完成周期后会把衣物从脏变为干净且湿，晾衣架会随时间把湿衣物变干，超市冷柜门打开会影响冷藏商品状态，医院病床使用后会生成清洁或床单补给需求。

\subsection{场景构建流程}
GraphWorld 采用“拓扑生成、房间填充、运行时化”的三阶段流程。第一阶段根据场景类型生成房间集合和连通关系，例如家庭包含卧室、厨房、浴室和客厅，医院包含大厅、诊室、药房、输液区和护士站，超市包含入口、货架区、生鲜区、冷柜区和收银区。第二阶段按照房间功能填充 anchor object、设备、容器和可移动物体，并补全支撑、包含和控制关系。第三阶段将静态图转换为可执行状态，建立 parent-of、relation-of、room-of、control edges 和 room edges 等索引，供运行时验证动作和更新图状态。

这一流程的设计目标不是追求几何真实性，而是生成具有可规划意义的符号世界。图中每个重要对象都应有明确的语义类型、状态字段、可执行动作和合理父节点；每个房间都应能够通过拓扑关系被机器人逐步探索；每类长期扰动都应对应至少一种可恢复路径。

\section{运行时系统}
\subsection{统一状态转移}
GraphWorld 的运行时每一步包含四个主要阶段。首先，系统根据当前分钟数、日期和 NPC 角色日程生成 human event。其次，机器人基于局部观察获得可见节点、可见房间、置信度和合法动作候选。然后，机器人动作由 validator 检查并由 transition rules 执行；随后 human event system 根据事件前置条件和效果模板更新人类位置、物体位置和节点状态。最后，environment system 推进设备周期、自然干燥、腐烂和世界时钟，并同步运行时动态边。

当前低层动作空间包括
\[
\{\texttt{move},\texttt{pick},\texttt{place},\texttt{press},\texttt{open},\texttt{close},\texttt{brush},\texttt{fold},\texttt{dump}\}.
\]
动作并非只检查语法合法性，还要满足世界规则。例如关闭容器不能直接取放物体，设备门未关闭时不能启动，布类物体不能通过普通 \texttt{brush} 直接洗干净，垃圾桶只能接收腐烂或烧焦食物，装水杯子必须在水槽执行 \texttt{dump}。这些规则保证评分不能被不符合常识的捷径绕过。

\subsection{NPC 作为世界动力学}
GraphWorld 中的 NPC 不是对话角色，而是持续扰动世界状态的内部生成器。每个 NPC 具有 role、persona、当前位置和当前活动。家庭场景中 resident 会起床、穿衣、洗漱、吃饭、离家、回家和睡觉；医院场景中 patient、nurse 和 doctor 分别执行挂号、候诊、叫号、看诊、开药、送药、输液、换床单和补耗材等事件；超市场景中 customer、cashier 和 stocker 会进入商店、取购物车、选购商品、结账、离店和理货。

每个事件由前置条件和效果组成。前置条件用于判断人类活动是否顺利，例如病人是否能拿到处方、护士是否有药品可送、顾客是否能完成结账。效果则把活动影响写回图状态，例如移动物体、改变清洁度、改变填充度、移动 actor 或生成补给需求。由此，任务不是人工静态列举出来的，而是在 NPC 使用世界的过程中自然出现。

\subsection{局部感知与事件日志}
机器人并不直接读取全图。Perception 模块根据机器人所在房间、相邻房间、结构门开闭和设备门开闭生成局部观察，并维护 room-level confidence。打开结构门会扩展可见房间；打开设备门会暴露设备内部物体。这使机器人需要通过移动和开门逐步获得信息，而不是在全知状态下选择动作。

所有机器人动作、人类事件、动作失败和定时转移都会写入 event log。该日志既用于回放，也用于计算人类事件成功率。每个实验会输出 replay、metrics CSV、TensorBoard scalar 和 summary JSON，便于复现实验和观察长期曲线。

\section{技能条件化机器人 Agent}
\subsection{问题动机}
在持续演化世界中，机器人面临的难点不是单步动作空间，而是长期任务语义与局部动作候选之间的错配。``把物品放回原位'' 可以由当前父节点与初始父节点的差异定义；但洗衣、倒垃圾、清空杯子、医院补药和清洁病床包含隐含流程，不能等价为一次 \texttt{brush} 或 \texttt{place}。如果每一步都让语言模型从候选动作中重新选择，模型容易被局部合法动作吸引，出现重复开关门、过早收纳、错误清洁方式或目标切换。

\subsection{技能表示}
为此，我们引入轻量级技能库。每个技能由触发条件、完成目标和阶段化子流程组成：
\[
\text{skill}=(\text{name},\text{trigger},\text{goal},\{\text{phase}_i\}_{i=1}^{n}).
\]
家庭场景包括 \texttt{laundry\_clothes}、\texttt{dispose\_food} 和 \texttt{empty\_cup}。医院场景包括补充处方单、补药、归还冷藏药品、清理医疗废物、收集脏床单、补充干净床单、归还轮椅、清洁候诊区和清洁诊疗床等技能。技能不是完整符号规划器，而是可读、可扩展的常识流程，用于在图状态上定义长期目标和阶段偏置。

以洗衣为例，技能阶段包括 \texttt{wash\_load}、\texttt{start\_washer}、\texttt{dry}、\texttt{fold} 和 \texttt{store}。系统根据图状态刷新当前 phase：脏衣物不在洗衣机中时需要装入，已在洗衣机中且设备未运行时需要关闭并启动，湿衣物需要晾干，干燥未折叠时需要折叠，已折叠但未入柜时需要收纳。

\subsection{决策流程}
机器人每一步的 prompt 包含五类信息：机器人状态、全局规则、active goal、相关技能和合法动作候选。语言模型输出 high-level task 与 low-level action index；随后系统用基于图状态的 ranker 检查该动作是否推进 active goal。如果当前 active goal 有明确阶段，而模型选择的动作明显偏离，ranker 会把动作拉回更能推进阶段的合法候选。这样，语言模型仍负责语义判断和目标选择，图引擎负责动作合法性、阶段一致性和长期目标跟踪。

\section{评价指标与实验设置}
\subsection{矩阵化评价}
GraphWorld 使用三类分数衡量长期表现。状态分数比较当前状态矩阵 $S_t$ 与稳态目标 $S_w$，关注清洁、开闭、湿度、腐烂、填充度、设备状态等是否健康。空间分数比较当前关系矩阵 $R_t$ 与稳态关系 $R_w$，关注可移动物体是否回到合理位置。人类事件分数统计 NPC 日程事件的成功率，衡量机器人是否支持了人的正常活动。综合分数为
\[
\text{score}=0.45\,\text{state}+0.35\,\text{spatial}+0.20\,\text{human}.
\]
分数按时间累计记录。坏状态持续越久，长期曲线受到的影响越大；机器人后续完成恢复任务时，instant score 会改善，累计分也会逐步回升。

\subsection{实验设置}
我们在五个代表性场景上运行 1600 step 实验：家庭场景包含 1 个 resident；医院、超市、办公室和工厂各包含 3 个 NPC，分别覆盖病人/护士/医生、顾客/收银员/理货员、员工/经理/访客、工人/质检员/维修员等角色。无机器人基线只运行 NPC 和环境转移；机器人设置加入 1 个机器人，使用 Qwen3.5-9B 作为语言模型。

\subsection{结果分析}
家庭场景中，single-round 方法将综合分从 0.7802 提升到 0.8918，主要收益来自空间分数从 0.6049 提升到 0.8743。这说明家庭任务较规则，给定 active goal 和技能后，一次 LLM 调用已经能够维护洗衣、餐具、垃圾和物体归位等闭环。

医院场景中，goal-review 明显优于 single-round：single-round 的综合分为 0.6698，甚至低于无机器人基线，而 goal-review 达到 0.8473。医院任务具有更强流程依赖，处方单、药品、冷藏药、床单和医疗废物都需要按阶段完成。将“是否继续当前目标”和“下一步做什么”拆开，有助于机器人坚持长链闭环。

超市、办公室和工厂展示了空间恢复的重要性。超市中，购物车、冷柜、货架和收银区被维护后，人类事件分数从 0.5629 提升到 0.9934。办公室中，文件、杯子、会议室和工位归位使空间分数从 0.3414 提升到 0.9681。工厂中，安全装备、零件箱、工具和质检记录构成物流闭环，single-round 将空间分数提升到 0.9858。

Reactive 方法提供了一个重要反例。它在家庭、医院和工厂中低于无机器人基线，说明“局部可见图 + 合法动作候选”并不足以让语言模型自发形成长期维护策略。GraphWorld 因而不只是比较分数高低，也能暴露 agent 的失败类型：局部 affordance 偏置、目标保持不足、闭环完成不足以及对人类活动时间窗口不敏感。

\section{局限与下一步}
当前 GraphWorld 仍是一个研究原型。首先，稳态目标仍主要基于初始状态，尚未充分支持 activity override。例如吃饭时餐具在桌上、出门时鞋在人身上、护士换床单时床单离开柜子，都可能是活动中的合理偏离；评价系统需要根据活动阶段动态切换合理目标。其次，多机器人协作仍不完整。系统已经支持多个机器人节点、独立 memory 和动作列表，但动作执行仍按顺序进行，缺少资源冲突检测、任务分配和事务式提交。第三，NPC 之间的依赖还比较模板化，医院和超市需要进一步加入队列、共享资源和跨角色事件依赖。最后，当前机器人主要依赖 LLM 与规则 ranker，尚未形成完整强化学习训练闭环。

下一步工作将优先补齐三件事：其一，完善超市机器人实验并加入办公室、工厂等场景；其二，引入全局任务池和机器人任务认领机制，避免多机器人重复追同一目标；其三，将评分中的稳态定义扩展为活动条件化目标，使评价更贴近真实持续世界。

\section{结论}
本文提出 GraphWorld，一种面向长时程具身智能的持续自演化图世界模型。GraphWorld 将场景图从静态表示扩展为运行时世界底座，把空间结构、对象状态、NPC 日程、环境时间和机器人动作统一在同一符号图中。通过技能条件化机器人 agent 和矩阵化长期评价，GraphWorld 能够研究机器人如何在不断变化的家庭、医院和超市中持续维护世界秩序。初步实验表明，机器人维护行为能够显著改善空间恢复、人类事件成功率和综合长期表现。我们认为，面向无限任务流的持久图世界是服务机器人从完成单次任务走向长期共处的重要基础。
